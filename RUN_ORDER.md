# Recommended Run Order

Run all notebooks from the repository root. The public repository does not include large raw public datasets, restricted source data, or generated result folders.

## 1. Prepare Environment

```bash
conda env create -f environment.yml
conda activate depprior
```

If R packages are installed outside Conda, use R 4.5.2 and install the packages listed in `scripts/collect_session_info.R`.

## 2. Prepare Input Data

Download public datasets from their original sources and place them in the expected local directories:

- `depmap/`: DepMap/CCLE model metadata, CRISPR gene effect, expression, and copy-number matrices.
- `data/TCGA/processed/`: TCGA-LUAD processed files generated by `code_r.ipynb`.
- `external/`: GEO and CPTAC matrices used for external reproducibility analyses.

See `data/external_data_manifest.csv` and `DATA_DICTIONARY.md` before running the notebooks.

## 3. TCGA/GDC Retrieval and Processing

Run:

```bash
jupyter notebook code_r.ipynb
```

Main expected outputs:

- TCGA-LUAD expression matrix.
- TCGA mutation summary and mutation matrix.
- TCGA copy-number matrix.
- TCGA methylation matrix.
- TCGA clinical/survival table.
- Multi-omics object for downstream analyses.

## 4. DepMap Modeling and Candidate Prioritization

Run:

```bash
jupyter notebook code.ipynb
```

Main expected outputs:

- DepMap LUAD aligned matrices.
- Train/test model benchmark summaries.
- Gene-wise AUROC, R2, and RMSE tables.
- High-signal candidate pool.
- DepScore-ranked candidate target set.
- TCGA candidate-set figures and tables.

## 5. Supplementary and External Reproducibility Analyses

Run:

```bash
jupyter notebook code_supplementary.ipynb
```

Main expected outputs:

- Threshold-sensitivity analyses.
- Alternative composite-score comparisons.
- Matched-size random gene-set benchmarks.
- GEO and CPTAC reproducibility analyses.
- Supplementary figures and tables.

## 6. Additional R Workflow Checks

Run only if the corresponding figure or table checks are required:

```bash
jupyter notebook running_r.ipynb
```

Main expected outputs:

- Additional survival summaries.
- Figure-generation checks.
- Exported survival-model performance tables.

## 7. Capture Final Software Versions

After the final successful run, capture the exact environment:

```bash
python scripts/collect_versions.py --out logs/python_versions.tsv
Rscript scripts/collect_session_info.R logs/R_sessionInfo.txt
```

The resulting `logs/` files may be committed if journal policy allows, or submitted as part of the reproducibility record.

## 8. Final Public Repository Check

Before resubmission, confirm that the GitHub repository contains:

- Code notebooks.
- README.
- Run order.
- Data dictionary and dataset manifest.
- Environment files and version-capture scripts.
- Random-seed/versioning note.
- Workflow diagram.
- Western blot source-data checklist or a link to the journal source-data package.
